Hub Information Driver Quick Reference

Commands

  • Configure - Sets the initial values for the base attributes
    • latitude
    • longitude
    • timeZone
    • zipCode
    • temperatureScale
    • id
    • name
    • data
    • zigbeeId
    • zigbeeEui
    • hardwareID
    • type
    • localIP
    • localSrvPortTCP
    • firmwareVersionString
    • uptime
    • locationName
    • locationId
    • hubModel
  • HIA Update - Allows the Hub Information Aggregation app to override the hmtl attribute
  • Initialize - Runs at hub start up to check polling status and re-run Configure
  • Reboot - If the preference is set to allow, this will reboot the hub (command is affected by Hub Security)
  • Refresh - Forces a run of the polling cycle which refreshes the attributes not covered by Configure
  • Shutdown - If the preference is set to allow, this will shutdown the hub (command is affected by Hub Security)
  • Update Check - Force an unscheduled check for a Firmware Update

Preferences

Most of these options require the driver to make a separate call to the hub gather the information, so it is recommended that you only select the data that you are actively interested. Additional note, the CPU load information is a 5 minute average so it recommended that polling be set at no less than 5 minutes (300 seconds).

  • Enable debug logging? - Creates Debug Entries in the HE Log for 30 minutes
  • Suppress Warn Level Logging - Don't create Warning log entries (most are simply informational)
  • Enable Temperature Polling - Request current CPU temperature (attributes: temperatureF, temperatureC, temperature)
  • Enable Free Memory Polling - Request available free memory (attribute: freeMemory)
  • Enable CPU Load Polling - Request CPU load information (attributes: cpuLoad, cpuPct)
  • Enable DB Size Polling - Request DB size (attribute: dbSize)
  • Enable Querying the cloud to obtain your Public IP Address? - Hit an external site to obtain the external IP address (attribute: publicIP )
  • Enable Display of Max Event/State Days Setting - Request the system defaults for maximum days event and state changes are logged (attributes: maxEvtDays,maxStateDays)
  • Enable HTML Attribute Creation? - Create a custom attribute for display on a tile when the polling runs (attribute: html)
  • Template file for HTML attribute - A file in the HE File Manager that contains information to build the html attribute
  • Force Update of ZWave Version Attribute - Request information on the ZWave Version on poll instead of just at start up (attributes: zwaveVersion, zwaveSDKVersion)
  • Never Run ZWave Version Update - Do not poll for ZWave information (the poll used to lock up C5, C4, and C3 hubs)
  • Check NTP Server on Poll - Request the server address used for time synchronization (attribute: ntpServer)
  • Check for additional Subnets on Poll - Check for secondary subnets (attribute: ipSubnetsAllowed)
  • Include Hub Mesh Data - Map data for any hubs participating in HubMesh (attributes: hubMeshData, hubMeshCount)
  • Include Expanded Network Data - HE v2.3.4.126 made available some addition network related data (attributes: connectType (Ethernet, WiFi, Dual), dnsServers, staticIPJson, lanIPAddr, wirelessIP, wifiNetwork)
  • Date/Time Format for Sunrise/Sunset - Simple Date Time Format string (attributes: sunrise, sunset)
  • Date/Time Format for Last Update - Simple Date Time Format string (attribute: lastUpdated)
  • Separator for Formatted Uptime - Character/string to place between the formatted time segments (attribute: formattedUptime)
  • Suppress data attribute if Zigbee is null - don't create attribute data if the Zigbee radio is off (attribute: data)
  • Remove unused attributes (Requires HE >= 2.2.8.141) - removes any attribute not currently in an update/polled status
  • Log all attribute changes - Create an Information level log entry for every attribute update, regardless of whether the value changes
  • Allow Hub to be shutdown or rebooted - Enable or disable the command for the driver to shutdown or reboot the hub
  • Hub Security Enabled - If Hub Security is turned on, enable this option to enter the Username and Password
    • Hub Security Username - Username to allow login to the hub
    • Hub Security Password - Password to allow login to the hub
  • Polling Rate (seconds) Default:300 - Frequency to query the hub
  • Poll rate (in seconds) for FW Update Check (Default:6000, Disable:0) - Frequency to check Hubitat for firmware updates
    • Use raw update notice - Use the update text returned from the query or the interpreted version (NO_UPDATE_AVAILABLE-->Current, UPDATE_AVAILABLE-->Update Available)
